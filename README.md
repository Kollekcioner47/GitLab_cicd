DevOps/GitLab CI/CD: Полный курс от установки до Production
Этот репозиторий содержит пошаговые инструкции для самостоятельного изучения DevOps-инструментов на примере развёртывания GitLab и построения конвейеров CI/CD. Курс проведёт вас от установки GitLab на виртуальную машину до создания полноценного микросервисного приложения с автоматическим деплоем, Review Apps и уведомлениями.

Цель курса:
Научиться настраивать GitLab, писать пайплайны CI/CD, работать с Docker, контейнеризировать приложения, внедрять проверки безопасности, организовывать деплой на удалённые серверы через SSH и создавать временные окружения для каждой ветки (Review Apps).

Содержание:
1_install.txt	Установка GitLab, генерация самоподписанного сертификата, настройка HTTPS
2_first_project.txt	Первый проект: статический сайт, Apache, простой pipeline
3_second_project.txt	Многостадийный pipeline: lint → build → staging → test → production (manual)
4_docker_runner.txt	Docker-раннер, Container Registry, деплой контейнеров
5_microservice_python_pg.txt	Микросервис на FastAPI + PostgreSQL, тесты с реальной БД
6_microservice_evolution1.txt	Веб-интерфейс на Go, версионирование образов, деплой staging
7_microservice_evolution2.txt	Кэширование зависимостей, ускорение сборки, сканирование уязвимостей (Trivy, SAST)
8_ssh_revewapps_secrets_etc.txt	Полный цикл: CRM-приложение, SSH-деплой, Review Apps, JUnit-отчёты, уведомления по email
Предварительные требования
Виртуальная машина (рекомендуется Debian или Ubuntu) с доступом в интернет.
Минимум 4 ГБ RAM, 2 ядра CPU, 20 ГБ свободного места.
Базовые навыки работы с командной строкой Linux, Git, Docker (желательно).

Для практик 5–8 потребуется дополнительная ВМ (или второй сервер), на котором будет разворачиваться приложение (можно использовать ту же машину, но с разными IP).

Как использовать
Склонируйте репозиторий на свою ВМ:
git clone https://github.com/Kollekcioner47/GitLab_cicd.git
cd devops-course
Открывайте файлы по порядку (от 1_install.txt до 8_...) и выполняйте инструкции.
Каждый файл – это законченная практика. Вы можете останавливаться после любой из них, но для полного понимания рекомендуется пройти последовательно.
Для практик 5–8 скопируйте код приложений из инструкций в соответствующие файлы (всё подробно описано внутри).
Примечание: Исходный код проектов (simple-site, notes-api, mini-crm) не включён в репозиторий, так как вы создадите его сами по инструкциям. Это даст вам практику написания кода и полного цикла разработки.

Основные технологии, которые вы освоите
GitLab (установка, конфигурация, самоподписные сертификаты)
GitLab CI/CD (пайплайны, стадии, артефакты, environments, переменные)
Docker (Dockerfile, образы, registry, docker-compose, кэширование)
Python (FastAPI, SQLAlchemy, pytest, TestClient)
Go (прокси-сервер, многоэтапная сборка)
Базы данных (PostgreSQL, миграции через SQL-скрипты)
Безопасность (Trivy, SAST, управление секретами)
Деплой (SSH, Docker socket binding, Review Apps, очистка окружений)

Результаты
После прохождения всех практик вы сможете:
Самостоятельно поднять GitLab в закрытой сети (on‑premise).
Написать CI/CD пайплайн с параллельными тестами, линтингом, сборкой образов, сканированием уязвимостей.
Развернуть многоконтейнерное приложение на удалённом сервере по SSH.
Создавать изолированные Review Apps для каждой ветки (Merge Request).
Настроить уведомления о результатах деплоя по электронной почте.
