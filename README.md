DevOps и GitLab CI/CD: Практический курс от установки до Production
Данный репозиторий содержит комплект последовательных практических работ по изучению современных DevOps-подходов на базе GitLab CI/CD. В ходе выполнения заданий слушатели проходят полный путь от развёртывания собственного GitLab-сервера до построения производственного конвейера непрерывной интеграции и доставки с использованием контейнеризации, автоматизированного тестирования, проверки безопасности и деплоя приложений.
О курсе
Курс ориентирован на системных администраторов, DevOps-инженеров, разработчиков и специалистов по сопровождению ИТ-инфраструктуры, желающих получить практический опыт построения CI/CD-процессов в собственной среде.
В рамках обучения рассматриваются:
•	развёртывание и настройка GitLab;
•	организация процессов непрерывной интеграции и доставки (CI/CD);
•	контейнеризация приложений с использованием Docker;
•	работа с GitLab Runner и Container Registry;
•	автоматизация тестирования и развёртывания;
•	управление секретами и безопасностью;
•	создание временных окружений (Review Apps);
•	деплой приложений на удалённые серверы через SSH.
Содержание курса
1_install.txt	Развёртывание GitLab, настройка HTTPS и самоподписанных сертификатов
2_first_project.txt	Первый проект: статический веб-сайт и базовый CI/CD-конвейер
3_second_project.txt	Многоэтапный pipeline: lint → build → staging → test → production
4_docker_runner.txt	Использование Docker Runner, Container Registry и контейнерного деплоя
5_microservice_python_pg.txt	Разработка и тестирование микросервиса FastAPI с PostgreSQL
6_microservice_evolution1.txt	Развитие проекта: веб-интерфейс на Go и автоматический деплой
7_microservice_evolution2.txt	Оптимизация сборки, кэширование зависимостей и контроль безопасности
8_ssh_revewapps_secrets_etc.txt	Production-подходы: SSH-деплой, Review Apps, отчётность и уведомления

Исходный код демонстрационных приложений не включён в репозиторий. В процессе выполнения практических работ слушатели самостоятельно создают необходимые проекты, что позволяет получить дополнительный опыт разработки и сопровождения программного обеспечения.

Требования к стенду
Для выполнения практических работ рекомендуется подготовить следующую инфраструктуру:
•	Виртуальная машина под управлением Debian или Ubuntu;
•	Не менее 4 ГБ оперативной памяти;
•	Не менее 2 виртуальных процессоров;
•	Не менее 20 ГБ свободного дискового пространства;
•	Доступ в Интернет для установки пакетов и контейнерных образов.
Для практических работы №8 рекомендуется использовать дополнительный сервер или виртуальную машину, предназначенную для развёртывания приложений.
Порядок прохождения
1.	Клонировать репозиторий:
git clone https://github.com/Kollekcioner47/GitLab_cicd.git
cd GitLab_cicd
2.	Последовательно выполнять инструкции, начиная с файла 1_install.txt.
3.	Каждая практика является самостоятельной и может использоваться отдельно.
4.	Для получения полного набора компетенций рекомендуется выполнять все работы в указанном порядке.

Результаты обучения
После завершения курса слушатель сможет:
•	Разворачивать и администрировать GitLab в локальной или корпоративной инфраструктуре.
